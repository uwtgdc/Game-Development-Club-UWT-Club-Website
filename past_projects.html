Game Development Club @ UWT's Game Engine

A 2D video game engine written in C++ by the UWT Game Dev.The UWT Game Dev engine is a project meant to span the lifetime of the UWT Game Dev club. Built in order to advance our skills in game development as well as C++ (a staple in the industry), it leverages the library SFML as well as C++ in order to create a working video game engine for use in 2D titles. Starter code provided by Justin Goding in addition to Professor Dave Churchill at Memorial University.

SFML Library

Features of the Game Development Club @ UWT's Game Engine

- Basic 2D Rendering

- Implementation of basic ECS usage, a staple in the video game industry

- 2D Collision

- Simple AI

- Input

Past Competitions

Throughout the club’s lifetime, one thing that has always been relevant is the club hosting competition based growth of and showcasing of programming and game development skills opportunities. These have taken the form primarily of Game Jams for the Game Development Club @ UWT.

Typically, these submissions can be toward a specific theme and are made in a timeframe of anything toward a few days to a month. Additionally, in most cases, the themes for the events are decided by primarily the club, the club officers, or anyone interested in picking a theme for the competitions.

The following is a Breakdown of the Event Types:

Game Jam: An event where individually or as a team to create a game within a certain timeframe.

Asset Jam: An event where individually or as a team you are creating an asset, script, or even a game within a certain timeframe.
